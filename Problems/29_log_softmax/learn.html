Solve Numerical Instability of Softmax

While Softmax is generally well-behaved, it can suffer from numerical instability issues, particularly when dealing with large input values.

Overflow: Exponentiating large numbers can lead to values exceeding the representable range of floating-point numbers, resulting in overflow errors.

Underflow: Conversely, exponentiating very small numbers can result in values approaching zero, causing underflow errors.

These numerical issues can lead to incorrect results and hinder the training process.

Write a Python function that computes the softmax activation without numerical instability. The function should return the softmax values as a list, each rounded to four decimal places.
